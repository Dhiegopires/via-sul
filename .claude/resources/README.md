# Resources

Biblioteca de referência (não é um step sequencial — como `tokens/`, fica disponível em paralelo para consulta durante qualquer etapa).

```
resources/
├── README.md                          ← este arquivo
└── design-resources-library.md        ← ~400 recursos de design/UX/UI categorizados
```

## O que é

`design-resources-library.md` é a varredura completa (~400 recursos) de [adhamdannaway.com/design-resources-library](https://www.adhamdannaway.com/design-resources-library), organizada nas mesmas categorias do original. Coletado em 2026-06-29.

## Como isso se conecta ao processo

Os recursos mais relevantes de cada categoria já estão linkados diretamente dentro do README de cada step do [`Full-website/`](../Full-website/README.md) (`1-discovery` a `7-post-delivery`), numa seção "Recursos de referência", com uma linha explicando por que aquele link importa naquela etapa específica. É lá que você consulta no dia a dia. Outros tiers, quando forem construídos, linkam pra cá do mesmo jeito.

Este arquivo é o backup completo: use quando precisar de algo que não está nos steps — outro icon set, outra fonte, outra referência de copy, etc.

Os links são externos e de terceiros — confira a validade antes de repassar a um cliente.
