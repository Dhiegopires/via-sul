## WHAT TO PRODUCE

Gere um arquivo `.md` salvo como `relatorio-[cliente]-[mes-ano].md` na pasta do projeto.

### Document structure:

**Header**
- Label: "Monthly Report"
- Title: [CLIENT NAME] — [MONTH YEAR]
- Meta row: Plan tier · Website · Prepared by VellumWire

**Section 1: Traffic**
Table with 3 rows: Sessions, Users, Bounce Rate.
Columns: Metric / This Month / Last Month / Change.
Change column: show percentage delta. Color: accent if positive, danger if negative, muted if flat.

**Section 2: Top Sources**
Table: Source / Sessions / % of total.
Three rows. No more.

**Section 3: Conversions**
Table: Event / Count.
Every configured conversion event gets a row. If the count is 0, show 0 — not a blank. List confirmed leads (`generate_lead`) and CTA clicks (`cta_click`) as separate rows — never combine or relabel a click count as the lead count. If confirmed leads is 0 while CTA clicks is not, say so plainly in Section 4 rather than letting the table imply the page isn't converting when the real gap may be downstream (the external booking tool, the offer) rather than the page itself.

**Section 4: What happened this month**
Card list. One card per item:
- Each edit made (Hold and Build)
- A/B test: what was tested, current status (Build only)
- Design delivery: what was delivered (Build only)
Keep each item to 1–2 sentences. No marketing language.

**Section 5: Heatmap insight (Build only)**
One callout box. One finding from Hotjar. What it shows, what it means for conversion. Max 3 sentences. If no Hotjar data was provided, omit this section entirely.

**Footer**
Left: "Monthly Report · [CLIENT NAME] × VellumWire"
Right: [MONTH YEAR]

## REGRAS

- Deltas positivos: prefixar com `+`. Negativos: prefixar com `-`.
- Métrica ausente: escrever `—` na célula + nota ao fim "Métrica indisponível este mês."
- Seção Hotjar/Heatmap (Build only): omitir completamente se nenhum dado Hotjar foi fornecido.
- Nenhuma linguagem de marketing: sem "resultados extraordinários", "crescimento acelerado", "insights acionáveis".
