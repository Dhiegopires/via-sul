# Visual Direction: AI Execution Prompt

You are defining the visual identity direction for a client's new website. The output is a structured `.md` file saved in the client project. Your task: make specific, justified creative decisions about colors, typography, and visual mood, and document them clearly so development can begin immediately after approval.

The goal of this document is to get explicit client approval on visual direction before any code is written. Every decision must be justified. "Modern and clean" is not a justification. "Warm dark background differentiates from all three main competitors who use light themes, and signals technical authority to the primary stakeholder" is a justification.

## WHAT TO DECIDE

Make specific decisions on each of the following. Do not hedge. Pick one direction and justify it.

**Decision 1: Background theme**
Dark or light? If dark: warm dark (brownish cast) or cool dark (blue cast)? If light: warm white or neutral white?
Justification must reference: competitor landscape, primary stakeholder expectations, product category.

**Decision 2: Primary accent color**
One color only, not a palette. A single accent that will define the brand identity. Must be:
- Distinctive from all competitor accent colors identified in market research
- Appropriate for the primary stakeholder's industry and trust level
- High contrast against the chosen background
State the hex value, the psychological association, and why it differentiates.

**Decision 3: Secondary accent (if needed)**
Optional. Only include if the product has two distinct segments or emotional tones that need separate visual signals. State both hex values and which segment each belongs to.

**Decision 4: Typography pairing**
Two fonts from Google Fonts. One display font for headlines, one body font for UI and copy.
Rules:
- Display font must have personality, not Inter, Space Grotesk, Plus Jakarta Sans, or any other ubiquitous SaaS default
- Body font must be highly readable at 15–16px
- The pairing must feel intentional, not random
- Justify each choice with reference to the brand's tone and the competitor landscape

**Decision 5: Visual mood**
Three adjectives that describe the overall feeling of the site. Then: one visual metaphor that captures the brand (e.g. "a precision instrument", "a trusted advisor", "raw data made beautiful"). This metaphor should inform every design decision downstream.

**Decision 6: What to avoid**
Based on the client's anti-references and competitor analysis: list 3–5 specific visual patterns to avoid. Be specific: not "avoid being generic" but "avoid the blue-gradient hero with floating dashboard screenshot that Competitor X and Y both use."

## OUTPUT FORMAT

Gere um arquivo `.md` salvo como `visual-direction-[cliente].md` na pasta do projeto. Cada decisão deve ter justificativa específica. Seções:

1. **Paleta de cores**: cor de fundo (hex + justificativa), cor de destaque primária (hex + justificativa), sistema completo — todas as superfícies, textos, bordas e status com hex e regra de uso
2. **Tipografia**: fonte display (nome + Google Fonts link + pesos + specimen com headline real do cliente), fonte corpo (nome + specimen), justificativa do par, escala tipográfica completa do display ao label com tamanhos em px
3. **Mood visual**: três adjetivos com explicação de uma frase cada + 3 implicações práticas específicas em decisões de design
4. **O que não somos**: 3–5 padrões visuais a evitar com razão específica + tabela de comparação com concorrentes (cores, fontes, estilo)
5. **Checklist de aprovação**: paleta (S/N + obs), tipografia (S/N + obs), mood (S/N + obs) + próximo passo: "Após aprovação, o desenvolvimento começa. Mudanças na direção visual após aprovação requerem revisão de escopo."

## TONE RULES

- Every decision must be justified with specific reasoning, never vague
- Reference competitors by name when explaining differentiation
- No AI language: "it is crucial", "worth noting", "leverage", "holistic"
- Write directly to the client: "We chose X because your three main competitors all use Y, which means X immediately sets you apart"
- The approval section must be clear about consequences: visual direction changes after approval cost time

---

## CRITICAL AI GUARDRAILS (HALLUCINATION-PROOFING)

To prevent AI hallucinations and generic "slop" output, you must strictly follow these rules:
1. **No Hallucinated Visual References**: Do not invent client reference URLs, likes, or dislikes. Only base choices on visual preferences explicitly pasted. If missing, write `[DATA NEEDED: client references]` styled in danger color.
2. **Justified Font Choices**: Choose specific Google Fonts that are available and loadable. Pair them intentionally based on client context. Do not invent custom font names.
3. **No AI Copy Patterns**:
   - Do not use em dashes (`—`) in copy.
   - Do not use emojis in written body text or headlines.
   - Do not use SaaS clichés ("Not only X, but Y", "Transform your business").
