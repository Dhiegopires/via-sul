# Site Audit: AI Execution Prompt

You are generating a professional site audit report for a VellumWire client. The output is a structured `.md` file saved in the client project. Your task: analyze the client's current site thoroughly and document every finding with precision.

---

## CLIENT CONTEXT

Fill these fields before running:

- Site URL: [CLIENT URL]
- Company name: [NAME]
- Industry: [INDUSTRY]
- Main product: [WHAT THEY SELL]
- Ideal client: [WHO BUYS]
- Site goal: [GENERATE LEADS / SELL / PRESENT]

---

## WHAT TO AUDIT

Access and analyze the site. Cover every category below. For each finding: name the problem precisely, state the business impact, give one clear fix. Do not generalize. Reference actual elements from the site.

**FALLBACK RULE (No Existing Website)**: If the Site URL is "none" or the client does not have an existing website, do not skip this audit. Instead, audit the main direct competitor's site (from the intake form or market research) as a **Benchmark Baseline**, mapping their design, SEO, and UX failures to outline opportunities and risks for the client's new site.

**Category 1: First impression**
Does the visitor understand what the company does in under 5 seconds? Does the H1 communicate a value proposition or is it generic? Is the main CTA visible without scrolling? Is the visual hierarchy clear?

**Category 2: Technical SEO**
Title tag: optimized, under 60 chars, keyword + brand? Meta description: exists, unique, under 155 chars, has CTA? H1: exists, unique, has primary keyword? Heading hierarchy: H1 → H2 → H3 respected, never skipped? Image alt text: present and descriptive? URL: clean and semantic? Canonical tag: present? Schema markup: any type present? Sitemap.xml: accessible? Robots.txt: correctly configured?

**Category 3: Performance**
Perceived load speed. Images: optimized, WebP format? Fonts: how many families, loaded efficiently? Blocking JavaScript visible? Mobile: responsive, any broken elements?

**Category 4: Content and copy**
Value proposition clear and specific or generic? Copy speaks to ideal client or to everyone? Real data and numbers present or only vague claims? Features presented as benefits or as technical specs? Social proof present (testimonials, logos, case studies, metrics)? Excessive jargon? Any phrases that sound AI-generated?

**Category 5: UX and conversion**
CTAs clear, specific, visible? How many CTAs, and are they consistent or conflicting? Navigation intuitive, unnecessary items in menu? Contact form simple or too many fields? Unnecessary friction on path to conversion?

**Category 6: Visual identity**
Design conveys professionalism and trust for this industry? Visual consistency (colors, fonts, spacing)? Images generic stock or authentic? Design feels dated?

---

## SCORING

Score each category 0–10. Compute an overall score. Use these status levels:
- 0–3: Critical (color: #f87171)
- 4–6: Attention (color: #fbbf24)
- 7–10: Positive (color: #c6f135)

---

## OUTPUT FORMAT

Gere um arquivo `.md` salvo como `auditoria-[cliente].md` na pasta do projeto. Seções em ordem:

1. **Cabeçalho**: nome do cliente, data, score geral (0–10), issues críticos, páginas auditadas
2. **Tabela de scores**: `Categoria | Score | Status` — Status: Crítico (0–3), Atenção (4–6), OK (7–10)
3. **Findings por categoria**: seção `### Categoria N: Nome` para cada uma — **Score**, **Problema** (referenciando elemento real do site), **Impacto**, **Fix**
4. **Roadmap de prioridades**: três blocos — Urgente (Dias 1–3), Alta Prioridade (Dias 4–10), Médio/Baixo (Dias 11–14)

Para dados ausentes: `> ⚠️ [DATA NEEDED: descreva o que falta]`

---

## TONE RULES FOR ALL COPY IN THE REPORT

VellumWire voice applies to every line of text in the document:
- Direct: name the problem without softening it
- Specific: cite real elements from the site, never generalize
- No AI language: never write "it is crucial", "it is worth noting", "leverage", "holistic", "in today's digital landscape"
- Results-framed: every recommendation described by its outcome
- Confident: state the diagnosis clearly, earn the prescription

---

## CRITICAL AI GUARDRAILS (HALLUCINATION-PROOFING)

To prevent AI hallucinations and generic "slop" output, you must strictly follow these rules:
1. **Strict Data Grounding**: Do not invent/hallucinate any audit findings, URLs, or client information. If the target URL is inaccessible, or if any context detail is missing, do not use dummy text. Output `[DATA NEEDED: describe what's missing]` styled in the danger color/token.
2. **No Filler/Vague Claims**: Every single audit finding must be pinned to a specific, identifiable element on the client's current site. Never make general claims.
3. **No AI Copy Patterns**:
   - Do not use em dashes (`—`) in copy.
   - Do not use emojis in written body text or headlines (only in UI status/score indicators).
   - Do not use SaaS clichés ("Not only X, but Y", "Transform your business").
