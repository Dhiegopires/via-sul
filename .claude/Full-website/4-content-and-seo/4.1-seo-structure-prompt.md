# SEO Structure: AI Execution Prompt

You are generating the SEO structure document for a VellumWire client's new website. The output is a structured `.md` file saved in the client project. Your task: define precise SEO decisions for every page based on the inputs below.

Every decision must be specific and justified. A meta title is not "the client's name plus a keyword." It is exactly 58 characters, starts with the primary keyword, and ends with the brand name after a dash.

---

## CLIENT CONTEXT

- Company: [NAME]
- Domain: [DOMAIN]
- Industry: [INDUSTRY]
- Language: [pt-BR / en / other]

---

## INPUTS

**Approved sitemap with URLs and primary keyword per page:**
[paste here]

**Full keyword map from market research:**
[paste here]

**Approved conversion architecture (Step 2.3) — mandatory section sequence and formula stage per page:**
[paste here]

---

## WHAT TO PRODUCE

For every page in the approved sitemap, define the following. Do not skip any page.

**Per page:**

- **URL**: the exact final URL, clean and semantic
- **Meta title**: max 60 characters, primary keyword in first 40 characters, brand name at end after " | " or " — ". Count the characters. If over 60, shorten.
- **Meta description**: max 155 characters, contains primary keyword naturally, states the benefit clearly, ends with an implicit or explicit CTA. Count the characters.
- **H1**: one per page, contains primary keyword, max 8 words, reads as a headline not a label
- **H2 structure**: list all H2s for the page **in the exact order of the mandatory section sequence from Step 2.3** — one H2 per section in that sequence, each targeting a secondary keyword from the keyword map where applicable, and each labeled with the formula stage it corresponds to (e.g. "H2 3 — Agitate"). Do not reorder sections to improve keyword flow. If a keyword doesn't fit the locked section order, flag it instead of moving the section.
- **Schema type**: which JSON-LD schema applies: Organization, SoftwareApplication, Service, FAQPage, Article, BreadcrumbList, Person, or combination
- **Canonical URL**: the exact canonical href value
- **Internal links in**: which other pages should link to this page and with what anchor text
- **Internal links out**: which pages this page should link to and with what anchor text
- **Open Graph**: og:title and og:description (can differ slightly from meta title/description)

Also produce:

**Global SEO settings:**
- robots.txt content
- sitemap.xml structure with all URLs, changefreq and priority per page
- Base metadata applicable to all pages (og:locale, og:type, og:site_name)

**Schema JSON-LD blocks:**
For each page that needs schema, write the complete JSON-LD block ready to paste into the `<head>`. Use real data from the client context. Never use placeholder values.

---

## OUTPUT FORMAT

Gere um arquivo `.md` salvo como `seo-structure-[cliente].md` na pasta do projeto. Seções em ordem:

1. **Cabeçalho**: nome do cliente, data, total de páginas, total de keywords mapeadas
2. **Configurações globais**: robots.txt planejado, estrutura do sitemap.xml, metadados base — em blocos de código markdown
3. **Uma seção por página** na ordem do sitemap, cada uma com: nome e URL, meta title (com contagem exata de caracteres), meta description (com contagem exata), H1, H2s em ordem, keyword primária, keyword secundária, schema JSON-LD em bloco de código, tabela de links internos
4. **Tabela de validação**: todas as meta titles e descriptions — `Página | Texto | Chars` — flaggando ultrapassagens de 60 / 155 com `⚠️`

Para dados ausentes: `> ⚠️ [DATA NEEDED: descreva o que falta]`

---

## RULES

- Literal Character Counting: You must calculate the exact mathematical length of each generated meta title and meta description. List the text, then write its character count. Do not estimate; count every letter, space, and punctuation mark literally.
- Every H1 must be unique across all pages. No two pages share an H1.
- Every page must have a different primary keyword. No keyword cannibalization.
- H2 order must match Step 2.3's mandatory section sequence exactly. SEO structure never overrides conversion architecture.
- Schema JSON-LD must use real values, never "Your Company Name" or "example.com"
- Internal link anchor text must be descriptive, never "click here" or "learn more"
- No AI language in any copy within the document

---

## CRITICAL AI GUARDRAILS (HALLUCINATION-PROOFING)

To prevent AI hallucinations and generic "slop" output, you must strictly follow these rules:
1. **Precise Length Validation**: You must programmatically calculate the exact length of each generated meta title (max 60 chars) and meta description (max 155 chars). If a text is too long, rewrite it until it is within limits. Do not estimate character counts.
2. **Schema Grounding**: Do not write template JSON-LD. Generate valid, fully completed JSON-LD schema using the client name and domain provided. If domain is missing, write `[DATA NEEDED: client domain]`.
3. **No AI Copy Patterns**:
   - Do not use em dashes (`—`) in copy.
   - Do not use emojis in written body text or headlines.
   - Do not use SaaS clichés ("Not only X, but Y", "Transform your business").
