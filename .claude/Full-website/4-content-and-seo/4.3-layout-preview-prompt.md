# Layout & Composition Preview: AI Execution Prompt

You are building a structural preview of a VellumWire client's priority pages. This is not a visual design and not the final site. It exists to get the client's sign-off that the page as assembled actually delivers the conversion architecture approved in Step 2.3 — section order, CTA placement, attention ratio — before Claude Code builds the real page in Step 5.

Why this exists: by this point, the client has approved the conversion architecture (Step 2.3), the visual direction (Step 3), and the copy (Step 4 Phase B). What they have never seen is those assembled into an actual page. The first time anyone sees the real composition is currently when Claude Code has already built it — too late to catch a structural problem cheaply, and the point at which section order silently drifts from what was locked in Step 2.3. This prompt closes that gap. It is a verification step, not a design step: the persuasion decisions were already made in Step 2.3. This step confirms they survived being assembled.

## WHAT TO PRODUCE

For each page being previewed, build a vertical stack of labeled, neutral wireframe blocks — one block per section, in the exact order defined in Step 2.3's mandatory section sequence. Each block must show:

- **Section label**: the section's name, its position in the page, its formula stage, and its conversion lever, all from Step 2.3 (e.g. "Section 3 of 7 — Social Proof — Formula stage: Solve — Lever: anxiety")
- **Real copy**: the actual approved H1/H2/eyebrow/body/CTA text from Step 4, not lorem ipsum and not paraphrased. If this preview reveals a section reads oddly once placed in sequence, that is exactly the kind of problem this document exists to catch — flag it in a note, don't quietly rewrite it.
- **Relative emphasis**: one of Large / Medium / Small, reflecting how much visual weight this section should carry based on its role (a hero gets Large, a single supporting stat block gets Small). Use the DS type scale to size the block's heading proportionally, but do not apply brand colors, imagery, shadows, or decorative styling — flat borders and neutral gray only. The point is to judge hierarchy and rhythm, not finished visual polish.
- **Internal link note**: which other page(s) this section links to, per Step 2's internal linking structure, if any

After all sections for a page, produce three checks, not just a section count:

1. **Section count**: total sections, flagging if the page exceeds 9 (usually means two pages got merged, or the sitemap needs a second look).
2. **CTA fold check, mobile-first**: mark which section block sits within a simulated first viewport at 375px mobile width first (use the DS spacing/type scale to estimate block height) — this is the primary check, run it before desktop, not after. Then confirm the same at a common desktop width. State plainly whether the CTA that Step 2.3 requires to be visible without scrolling (primary, or the lower-commitment secondary if the page's awareness level requires agitation first) actually lands inside that first viewport once real copy and DS proportions are applied, on mobile specifically. If it doesn't, this is not a note for Open Questions — it is a fail, and the page does not proceed to Step 5 until the section order or copy length is fixed to make it true.
3. **Attention ratio check**: recount, using the real assembled blocks, every action competing with the primary CTA within the mandatory sequence (secondary CTAs, internal links, repeated CTAs). Confirm the count still matches the budget approved in Step 2.3. If assembling real copy revealed more competing actions than Step 2.3 accounted for, flag it.

**Note on JS-driven heroes:** this document is a static wireframe — it cannot simulate a carousel's default state, a segment picker, or any other on-load JS behavior that might swap which CTA is showing. If the approved hero design includes any such interactive element, say so explicitly in the Open Questions panel and flag that the fold check must be re-verified in the real browser once built (Step 6.8, CV2) — a static preview passing this check is not proof the live JS-driven hero will.

## TONE RULES

- Do not let this turn into a visual design exercise. If you find yourself picking colors or imagery, stop — that already happened in Step 3, and re-litigating it here defeats the purpose.
- Do not let this turn into a persuasion-strategy exercise either. If you find yourself reassigning formula stages or reordering sections, stop — that already happened in Step 2.3. This step verifies the assembly, it does not redesign the architecture.
- Flag composition and CTA-fold concerns honestly even if you wrote the copy yourself in Step 4. The reason this step exists is that copy and structure can each look fine in isolation and still combine badly, or push the CTA below the fold once real copy length is applied.

---

## RULES

- Section order must match Step 2.3's mandatory sequence exactly. If you think the order should change, that is not a call this step gets to make — flag it in the Open Questions panel and route it back to Step 2.3, don't silently reorder.
- Copy must match Step 4's approved document exactly. Do not paraphrase, shorten, or "clean up" a sentence because it looks long in a block. If a section's real copy is long enough to push the required CTA out of the first viewport, that is a finding to report, not a reason to trim the copy yourself.
- Only preview the pages listed in CLIENT CONTEXT. If multiple secondary pages share the same template (e.g. several service pages with an identical section structure), say so explicitly instead of previewing each one — the client is approving a pattern, not auditing every page.
- The CTA fold check is a Blocker, not a note. A page does not get sign-off with a failed fold check, regardless of how good the rest of the composition looks.

---

## CRITICAL AI GUARDRAILS (HALLUCINATION-PROOFING)

1. **No invented sections.** Every block must trace to a section listed in Step 2.3's mandatory sequence. If Step 4's copy contains a section that isn't in the Step 2.3 list (or vice versa), flag the mismatch in Open Questions instead of resolving it silently.
2. **No invented copy.** Every word inside a block must come from the Step 4 document. If a section has no copy yet, the block says `[COPY NEEDED]` in danger color — do not fill it with placeholder text.
3. **No styling drift.** Do not apply DS brand colors, shadows, or imagery inside the preview blocks even if doing so would "look nicer." That defeats the document's purpose.
4. **No silent architecture changes.** Do not reorder sections, reassign formula stages, or change CTA placement to make the fold check pass. If it fails, report the failure and the exact section/copy causing it — the fix happens by revising Step 2.3 or trimming approved copy through a real revision, not by the AI quietly rearranging blocks in this document.
